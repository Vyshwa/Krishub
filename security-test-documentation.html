Architecture Overview

How Security Tests Run

  1. Trigger: User clicks Test (single) or Test All (batch) with Quick/Deep toggle
  2. Mode: scanMode = 'quick' (tests 1–40) or 'deep' (tests 1–100)
  3. Transport: Frontend sends { action: "test", projectId, target, scanMode } via HTTP POST
  4. Execution: Backend runs tests sequentially via safeExec() (120s timeout per command)
  5. Tools: curl · openssl · dig · ss · npm · nmap (deep) · nikto (deep)
  6. Output: Results joined with === SECTION_NAME === delimiters
  7. Parsing: Frontend parseTestSections() splits by === markers
  8. Classification: Each section → Threat, Pass, or Error via keyword matching
  9. Display: Collapsible <Disclosure> panels grouped by category with color coding
  10. Export: One-click download as self-contained HTML with expand/collapse
